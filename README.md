# KCG Ventures ERP – Build Tutorial

This repository contains your personalized, multi-page tutorial for building the **KCG Ventures ERP** web application using **Next.js**, **InstantDB**, and **Vercel**.

The tutorial site itself is a static HTML/CSS/JS website that you can view locally or host on **GitHub Pages**.

---

## Files in This Tutorial

- `index.html` – Page 1: Introduction & concepts (what you are building and how web apps work)
- `setup.html` – Page 2: Prerequisites & setup (accounts, InstantDB App ID, Node.js, Cursor)
- `build.html` – Page 3: Building your app (includes the fully customized build prompt)
- `deploy.html` – Page 4: Deployment to Vercel
- `reference.html` – Page 5: Next steps & troubleshooting (collapsible FAQ)
- `styles.css` – Shared custom styles
- `scripts.js` – Shared interactivity (copy buttons, accordions, smooth scroll, fade-in)
- `README.md` – This file

## Viewing the Tutorial Locally

You can open this tutorial directly from your computer without any extra tools:

1. Locate this folder on your machine.
2. Double-click `index.html` (or right-click and choose “Open with” → your browser).
3. Use the navigation bar at the top to move between:
   - Introduction
   - Setup
   - Build
   - Deploy
   - Reference

> Tip: For the best experience, keep this tutorial open in a browser tab while you work in **Cursor** in another window.

## Hosting on GitHub Pages (Recommended)

Hosting this tutorial on GitHub Pages makes it easy to access from any device and share with collaborators.

### Step 1: Create a GitHub Repository

1. Go to [https://github.com](https://github.com) and sign in.
2. Click the **+** button in the top right → **New repository**.
3. Choose a repository name, for example: `kcg-ventures-erp-tutorial`.
4. Select **Public**.
5. You do **not** need to add a README or .gitignore here (this folder already contains them if needed).
6. Click **Create repository**.

### Step 2: Upload the Tutorial Files

1. On your new repository page, click **“uploading an existing file”** (usually linked in the middle of the page).
2. In your file explorer, select and drag all of the tutorial files into the upload area:
   - `index.html`
   - `setup.html`
   - `build.html`
   - `deploy.html`
   - `reference.html`
   - `styles.css`
   - `scripts.js`
   - `README.md`
3. Scroll down and click **Commit changes**.

### Step 3: Enable GitHub Pages

1. In your repository, click the **Settings** tab.
2. In the left sidebar, click **Pages**.
3. Under **Source**, choose:
   - **Deploy from a branch**
   - Branch: `main`
   - Folder: `/ (root)`
4. Click **Save**.

After a minute or two, GitHub will build and publish your site.

### Step 4: View Your Live Tutorial

Your tutorial will be live at a URL like:

```text
https://YOUR-USERNAME.github.io/REPOSITORY-NAME/
```

For example:

```text
https://alex-founder.github.io/kcg-ventures-erp-tutorial/
```

You can now:

- Open the tutorial from any device.
- Share the link with collaborators, mentors, or future you.
- Refer back to it whenever you build similar apps.
