Ask the AI to build KCG Ventures ERP

On this page you'll create a new project in Cursor and use a carefully designed build prompt to generate a working Next.js + InstantDB version of your KCG Ventures ERP.

1. Create your project folder

First, you'll create an empty folder on your computer where all of the app's files will live.

  1. Create a new folder on your computer. Name it something like kcg-ventures-erp (no spaces – use dashes).
  2. Open Cursor.
  3. In Cursor, go to File → Open Folder and select the new folder you just created.
  4. You should see the folder name appear in the sidebar. It will be empty for now – that's expected.

2. Use the build prompt

Now you'll give Cursor a complete, detailed description of the app you want it to build. The AI will create:

  • A Next.js project with the App Router and TypeScript.
  • An InstantDB schema and permissions files.
  • Pages and components that match your KCG Ventures ERP workflow.

You don't need to modify this prompt beyond pasting your InstantDB App ID into the two marked places.

How to run this prompt in Cursor
  1. In Cursor, open the project folder you just created.
  2. Open the Composer (Cmd+I on Mac, Ctrl+I on Windows).
  3. Copy the entire prompt below.
  4. Paste it into the Composer and replace every occurrence of YOUR_APP_ID_HERE with your actual InstantDB App ID.
  5. Press Enter and let Cursor work. This may take a few minutes.

## Tech Stack (DO NOT DEVIATE)
- Next.js 14+ with App Router (not Pages Router)
- TypeScript

## InstantDB Configuration
My InstantDB App ID is: YOUR_APP_ID_HERE

Initialize InstantDB in a file at `src/lib/db.ts` like this:
```typescript
import { init } from '@instantdb/react';
import schema from '../../instant.schema';

const APP_ID = 'YOUR_APP_ID_HERE';

export const db = init({
  appId: APP_ID,
  schema,
```

## App Description
Build an app called **KCG Ventures ERP** for the shared BotanIQals + MiniLeaf facility. It is a production planning, inventory, and costing ERP that replaces complex Google Sheets workflows. The app should:
- Plan production cycles and tray schedules.
- Check freeze-dryer capacity feasibility.
- Track microgreen yields and inventory with cycle counts.
- Manage products and Bills of Materials (BOMs).
- Project material costs, revenue, and gross profit for each cycle.

Customer/POS flows (orders, invoicing, payments, shipping) and full accounting are out of scope.

## Authentication
Implement user authentication using InstantDB's Magic Code flow:
1. User enters their email.
2. InstantDB sends a 6-digit code to that email.
3. User enters the code to log in.
4. Use `db.useAuth()` to check login status.
5. Protect all routes that read or write production data so only authenticated users can access them.

When a user is not logged in:
- Redirect them to the `/login` page when they try to access protected pages.
- Show a simple, friendly message explaining that they need to log in to use the ERP.

- Inputs (from linked `production_targets` and calibration, BOM, and microgreens).
- Outputs should include:

  - Resource feasibility (inventory shortages).

### Freeze Dryer Feasibility Model (Important)

Use this model based on the idea/architecture docs:

1. Inclusive production window:
```ts
const production_days = Math.floor((end - start) / msPerDay) + 1;
```
Where `start` and `end` are dates and `msPerDay` is the number of milliseconds in a day.

2. From calibration:
```ts
const hours_per_cycle = cycle_time_hours + defrost_cleaning_hours;

const per_cycle_fresh_capacity =
  number_of_freeze_dryers *
  trays_per_machine_per_cycle *
  fresh_load_per_tray_g;

const total_available_hours = production_days * operating_hours_per_day;

const max_cycles_available = Math.floor(total_available_hours / hours_per_cycle);
```

3. Convert target dried requirement → fresh requirement:
```ts
const fresh_needed_g = dried_needed_g / dry_matter_fraction;
```

4. Compute cycles needed:
```ts
const cycles_needed = fresh_needed_g / per_cycle_fresh_capacity;
```

5. Feasibility check:
```ts
const feasible = max_cycles_available >= Math.ceil(cycles_needed);
```

Display for the user:
- Required cycles (`Math.ceil(cycles_needed)`).
- Available cycles (`max_cycles_available`).
- Approximate trays required (e.g. `Math.ceil(cycles_needed * trays_per_machine_per_cycle)`).
- Total hours required (`Math.ceil(cycles_needed) * hours_per_cycle`).
- A clear feasible / not-feasible status.

### Resource Feasibility (Inventory Sufficiency)

For each production cycle:
1. For each `production_targets` record (product + target_units), and for each `bom_lines` row for that product:
```ts
const required_qty = target_units * qty_per_unit;
```

2. For each inventory item, aggregate required quantity across all products:
```ts
const total_required_for_item = sum(required_qty_for_that_item_across_targets);
```

3. Compare to `quantity_on_hand`:
```ts
const shortage = Math.max(0, total_required_for_item - quantity_on_hand);
```

4. Output a shortages table:
- item name
- required
- on_hand
- shortage
- unit

### Cost, Revenue, and Gross Profit

Per production cycle:
```ts
const material_cost = sum_over_items(total_required_for_item * cost_per_unit);
const revenue = sum_over_products(target_units * sale_price_per_unit);
const gross_profit = revenue - material_cost;
```

Display both totals and per-product breakdowns.

### Tray Plan Generation (High-Level)

Based on:
- Microgreen parameters (soaking_required, germination_days, days_to_harvest, sow_rate_g_per_tray).
- Yield averages from `yield_entries` (fallback to simple defaults if there is not enough data).
- Cycle window.

Generate `production_plan_lines` that specify:
- Which microgreen each run uses.
- Run number and trays_this_run.
- Dates:
  - `soak_date` (if soaking_required; typically 1 day before sow).
  - `sow_date`.
  - `light_date` (sow_date + germination_days).
  - `harvest_date`.

You do not need to implement email sending in this phase; just generate and store the plan lines.

10. `/settings/notifications` (optional, UI only)

  - A toggle to enable/disable reminders.

## InstantDB Permissions
In `instant.perms.ts`, configure permissions so that:
- Only authenticated users can read/write data.
- Users can only read and write records that belong to them (based on links to `$users`).
- It is safe to use this app on the public internet without exposing other users' data.


4. Run your app locally

After Cursor finishes generating code, you're ready to see KCG Ventures ERP running on your own machine.

  1. In Cursor, open the integrated terminal (View → Terminal).
  2. Make sure the terminal is pointed at your project folder (it usually is, by default).
  3. Run the development server:
npm run dev

Wait until you see a line that includes something like http://localhost:3000.

What is localhost:3000?

localhost is a special address that means “this computer.” The :3000 part is like a specific doorway, or port, where your app is listening for requests.

When you type http://localhost:3000 into your browser, you're asking your computer: “Show me the web app that is currently running on port 3000.”

Open your browser and go to http://localhost:3000. You should see the homepage of your new KCG Ventures ERP app.

5. Fix issues with the AI's help

It's common for the first version of a generated app to have a few rough edges: TypeScript errors, missing imports, or other small issues. You don't need to debug these manually.

  1. If the terminal shows an error, select the error message text and copy it.
  2. If the browser shows an error screen, copy the error message from there instead.
  3. Open Cursor's Composer (Cmd+I / Ctrl+I).
  4. Paste the error and say something like: "I'm getting this error when I run `npm run dev`. Please fix it for me in this project."
  5. Let Cursor propose code changes. Review them briefly, then accept/apply the edits.
  6. Re-run npm run dev and check again.

This error → ask AI → apply fix loop is normal. Over time you'll understand more of what the AI is doing, but you can start using the app long before you understand every line of code.
