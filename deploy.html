Put KCG Ventures ERP on the internet

Right now your app only runs on your computer. In this step you'll push your code to GitHub and let Vercel deploy it to a live URL you can share with anyone.

1. What deployment actually is

When you run npm run dev, your laptop spins up a temporary web server at http://localhost:3000. The moment you close your terminal or shut your laptop, that server disappears.

Deployment means running your app on a server that is always on, connected to the internet, with a stable URL.

Mental model: moving from test kitchen to full restaurant

Think of localhost:3000 as a test kitchen in your garage – only you can eat there, and only while you're home. Deployment is like leasing a building in a busy area (Vercel) where the restaurant is open even when you're not there, and anyone with the address can walk in.

Vercel is designed specifically for apps like yours. Once you connect it to your GitHub repository:

  • It builds your app for production.
  • It deploys it to a live URL with HTTPS enabled.
  • It re-deploys automatically whenever you push new changes.

2. Push your code to GitHub

Vercel reads your code from GitHub. So the first step is to get your local project into a GitHub repository.

Create a new GitHub repository

  1. Go to github.com and sign in.
  2. Click the + icon in the top-right corner.
  3. Select New repository.
  4. Choose a repository name, for example kcg-ventures-erp.
  5. Set the visibility to Public.
  6. Important: do not add a README, .gitignore, or license from this screen – Cursor may already have created these locally.
  7. Click Create repository.
Connect your local project to GitHub

On the repository page you just created, GitHub shows a set of commands under “...or push an existing repository from the command line.”

Instead of typing them manually, you can ask Cursor to guide you:

  1. Copy the repository URL from GitHub (it ends in .git).
  2. In Cursor, open the terminal for your project folder.
  3. Open the Composer and tell Cursor something like: "Please help me push this project to GitHub. My repository URL is https://github.com/USERNAME/REPO-NAME.git".
  4. Let Cursor suggest and run the necessary git commands (initializing the repo, committing, adding the remote, and pushing).

When you refresh your GitHub repository page, you should see your full project tree: src, instant.schema.ts, instant.perms.ts, package.json, etc.

3. Deploy your app on Vercel

Now you'll connect Vercel to your GitHub repo and configure the one environment variable your app needs.

Connect the repository

  1. Go to vercel.com and sign in.
  2. From the dashboard, click Add New....
  3. Select Project.
  4. You'll see a list of your GitHub repositories. Find the one you just created for KCG Ventures ERP and click Import.

Add your InstantDB App ID as an environment variable

Before you click Deploy, you need to tell Vercel which InstantDB app to connect to. You'll do this using an environment variable.

What are environment variables?

Environment variables are a way to pass configuration values (like keys, IDs, or URLs) into your app without hard-coding them into the source files. They're especially useful when:

  • You have different values for development vs production.
  • You don't want to expose sensitive values in public code.

In this case, your InstantDB App ID is safe to share, but we still follow the pattern so your project matches real-world best practices.

  1. In the project import screen on Vercel, look for the section labeled Environment Variables.
  2. Add a new variable with:
    • Name: NEXT_PUBLIC_INSTANT_APP_ID
    • Value: your actual InstantDB App ID
  3. Click Add or the equivalent button to save it.
  4. Now click Deploy.

Vercel will now build your app. This usually takes 1–3 minutes. You can watch the build logs to see what's happening.

4. Visit your live ERP

When the build finishes successfully, Vercel will show a success screen with a URL, such as: 

https://kcg-ventures-erp-yourname.vercel.app

This URL is your app's new permanent address. Anyone with the link can open it in their browser, log in, and (subject to your permissions) use the system.

  • You can open it on your phone and test the mobile layout.
  • You can share it privately with collaborators or advisors.
  • You can keep iterating on the app by editing the code locally and pushing new commits to GitHub – Vercel will re-deploy automatically.
Automatic updates via GitHub

Any time you:

  • Make changes to the code in Cursor.
  • Commit those changes with Git.
  • Push them to the main branch on GitHub.

Vercel will notice the new commit, rebuild your app, and update the live URL, usually within a couple of minutes. You don't have to log into servers or run deployment scripts manually.

5. If deployment fails

Sometimes the first deployment doesn't work on the first try. That's normal. The key is to look at the error message and feed it to the AI.

  1. In Vercel, open your project and click on the failed deployment (it will be red or show an error icon).
  2. Scroll through the build logs. Look for the first place where something turns red or shows “Error”.
  3. Select the relevant error text and copy it. Focus on the core error, not the entire log.
  4. Go back to Cursor, open the Composer, and say something like: "Vercel deployment failed. Here is the build log error. Please fix my project so it can deploy successfully."
  5. Paste the error message after that sentence.
  6. Let Cursor propose code or config changes, apply them, push to GitHub, and try deploying again.

Over time, you'll start to recognize some of the common issues (missing environment variable, small TypeScript mismatch, etc.), but you don't have to start there.
