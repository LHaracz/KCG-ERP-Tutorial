Set up your accounts and tools

On this page you'll create the few accounts your app needs, install the tools on your computer, and prepare InstantDB so your KCG Ventures ERP can store real production data.

You'll only do this once per computer. After that, you'll reuse this setup for other apps too.

Create your online accounts

1. GitHub – where your code lives

What GitHub is: a website where developers store and share code. Think of it as a cloud-based filing cabinet for your projects.

Why you need it: your KCG Ventures ERP code will live in a GitHub repository. Vercel connects to that repository to deploy your app to the internet.

  1. Go to github.com.
  2. Click Sign up and create an account.
  3. Verify your email address when prompted.
  4. Choose a professional username – it will appear in your public URLs (for example: https://github.com/your-username).

2. Vercel – where your app will run

What Vercel is: a hosting service that takes your Next.js code and makes it available as a live website.

Why you need it: right now your app will run only on your laptop. Vercel runs it on servers so you, and others, can access it from anywhere.

  1. Go to vercel.com.
  2. Click Sign Up.
  3. Choose Continue with GitHub so Vercel can connect to your repositories.
  4. Authorize Vercel when GitHub asks for permission.
  5. The free Hobby plan is enough for this project and many future ones.

3. InstantDB – your backend & database

What InstantDB is: a backend service that stores your data and syncs it to the frontend in real time. It also provides user authentication.

Why you need it: your ERP needs a safe, central place to store calibration settings, yield logs, production batches, inventory transactions, and finished goods.

  1. Go to instantdb.com.
  2. Click Sign up and create an account.
  3. Verify your email address.
  4. Sign in to reach the InstantDB dashboard.

Create your InstantDB app

Now you'll create a dedicated InstantDB “app” that will hold all of your KCG Ventures ERP data.

  1. In your browser, go to https://www.instantdb.com/dash.
  2. Click Create App.
  3. Give it a name like KCG Ventures ERP or kcg-erp.
  4. Click to create the app.
  5. You'll see an App ID – a long string of letters and numbers, for example: 566ffee5-e773-4b68-a132-29678f8f6eea.
What is the InstantDB App ID?

Think of the App ID as the mailing address for your database. When your Next.js frontend wants to read or write data (for example, “create a new batch” or “update inventory”), it needs to know which InstantDB app it should talk to.

  • The App ID is not a password. It's safe to include in your frontend code.
  • InstantDB uses a permissions system to decide who can read and write which data.
  • You'll use this App ID in your build prompt so the generated app connects to the correct database.

Action: copy your App ID into a safe place (notes app, text file). You'll paste it into the build prompt on the next page.

Install Node.js and meet the terminal

What is Node.js?

Web apps are written in JavaScript. In production, most of that JavaScript runs in your users' browsers. But during development, you need to run tools on your own computer:

  • Starting the development server that shows your app at localhost:3000.
  • Installing packages like Next.js and InstantDB.
  • Building your app for deployment.

Node.js is the JavaScript engine that makes this possible. It lets your computer understand and run JavaScript outside the browser.

Install Node.js

  1. Go to https://nodejs.org.
  2. Download the LTS (Long Term Support) version. It is the stable, recommended option.
  3. Run the installer and follow the prompts.
  4. When it finishes, you can verify the installation in your terminal.
What is the terminal?

The terminal (also called the command line) is a text-based way to give instructions to your computer. Instead of clicking icons, you type short commands.

  • On macOS: open the Terminal app.
  • On Windows: open PowerShell or Command Prompt.

It can look intimidating at first, but in this tutorial you'll use only a few, very specific commands – and you can always ask the AI to explain what they do.

To confirm Node.js is installed, type this into your terminal and press Enter: 

node --version

If you see a version number (like v20.x.x), you're good to go.

Install and configure Cursor

What Cursor is and why we're using it

Cursor is a code editor with a built-in AI assistant. Instead of you writing JavaScript or TypeScript by hand, you'll describe what you want in plain English and let Cursor generate the code.

You are not trying to become a professional software engineer overnight. You're learning to direct software projects using AI – a very valuable skill for a business founder or operator.

Install Cursor

  1. Go to https://cursor.com.
  2. Download Cursor for your operating system.
  3. Run the installer and open Cursor.
  4. Sign in or create a Cursor account when prompted.

Give Cursor InstantDB as a “cheat sheet”

Cursor's AI has seen a lot of JavaScript and React, but InstantDB is relatively new. To help the AI generate correct code, you can plug InstantDB's official documentation directly into Cursor.

Think of this as handing your AI assistant a reference manual for your backend so it doesn't have to guess.

When to add docs to Cursor
  • Do add docs for newer tools like InstantDB, or when you see the AI guessing incorrectly.
  • Don't worry about adding docs for very standard tools (basic JavaScript, HTML, CSS) – the AI already knows those well.

Steps to add InstantDB docs in Cursor:

  1. Open Cursor.
  2. Open Settings (Cmd+, on Mac, Ctrl+, on Windows).
  3. Go to Features → Docs.
  4. Click Add Doc.
  5. Paste this URL: https://www.instantdb.com/docs
  6. Wait for Cursor to finish indexing (this can take a minute).

After this, when you ask Cursor to write code that uses InstantDB, it can look things up just like you would.

  • A Vercel account for hosting your live app.
  • An InstantDB account and App ID for your ERP's data.
  • Node.js installed and working.
  • Cursor installed and configured with InstantDB docs.
