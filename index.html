What you're building

KCG Ventures ERP is a focused production planning, inventory, and costing tool for the shared BotanIQals + MiniLeaf facility. Instead of wrestling with complex Google Sheets and Apps Script, you'll have a structured app that:

  • Stores a single, configurable freeze-dryer calibration profile (number of dryers, trays, cycle times, operating hours, load per tray, etc.).
  • Captures master data for microgreens, products, and Bills of Materials (BOMs) so planning is driven by real grow and cost parameters.
  • Logs yield data from individual trays so you can see rolling averages for fresh and dried output and feed those back into planning.
  • Maintains inventory ledgers and cycle counts for all materials, with an append-only adjustment log so you always know why quantity changed.
  • Lets you create production cycles, set product targets, and generate feasibility, shortages, cost/revenue, and tray schedule lines for operators to follow.

The goal is not to perfectly replicate every spreadsheet you have today. The goal is to build a real, working foundation that captures your most important workflows and can grow over time.

How modern web apps work

Before you build anything, it's worth building a solid mental model for what a web app actually is. This will make every future project feel less like magic and more like a system you understand.

Websites vs. web apps

A website is mostly static information – for example, a restaurant menu or a marketing page. Every visitor sees the same thing, and nothing is really “saved” per user.

A web app is interactive and personalized. Users can log in, create records, update data, and the app remembers who did what. Google Sheets, Notion, and banking portals are all web apps.

Your KCG Ventures ERP is firmly in the web app category because each production manager needs to log in, plan their own batches, and see their own data.

Why your app needs a backend

If your app only ran in the browser, any data would vanish the moment you closed the tab. That might be okay for a calculator. It is not okay for calibration settings, inventory, or batch history.

You need a central, persistent place where data lives – a “system of record” for your operation. That's the job of the backend and database.

Mental model: your app as a restaurant

This analogy will show up throughout the tutorial.

Imagine your production facility as a restaurant. The dining room is where customers sit, read the menu, and place orders. The kitchen is where food is prepared, scheduled, and stored. Customers never go into the kitchen—they only interact with what they can see and request.

  • Frontend (client) = the dining room interface. In your app, this is the set of pages users see in their browser: navigation, forms (e.g. “Create batch”), tables of inventory, etc.
  • Backend (server + database) = the kitchen and pantry. It stores calibration settings, yield logs, inventory ledgers, and batch records. Users never see raw database tables, they just see results.
  • API = the waiter carrying orders and plates. Whenever the frontend needs something (“create a batch”, “update inventory”), it sends a request to the backend through this API.

In this tutorial, you'll use tools that give you a ready-made kitchen and waitstaff. You focus on the “menu” and flows, not on building industrial equipment from scratch.

How this maps to your ERP

Concretely, here's how the pieces will line up for KCG Ventures ERP:

  • The frontend (dining room) is built with Next.js + React. It renders pages like “Calibration”, “Yield Logging”, “Batch Planner”, and “Inventory”.
  • The backend & database (kitchen) are provided by InstantDB. It stores your entities: calibration settings, yield entries, batches, inventory items, transactions, and finished goods.
  • You'll use user accounts so each person logs in and sees their own data. InstantDB's Magic Code login means no passwords to manage.

The tools you'll use

You'll use a small, carefully chosen toolset that plays nicely together and is friendly to beginners using AI assistance.

Cursor An AI-powered code editor where you describe what you want and it writes code. Your coding assistant and project co-pilot.
